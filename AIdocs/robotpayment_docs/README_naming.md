# ファイル名の重複を避ける命名ルール（統合フォルダ運用向け）

同一フォルダに仕様Markdownを全て格納する運用の場合、`01_overview.md` のような汎用名だと
他の仕様書（通常決済・自動課金・情報変更・停止など）と必ず衝突します。

## 推奨命名
`<docPrefix>_<nn>_<slug>.md`

- `docPrefix`：仕様書の種類を表す固定プレフィックス（例）
  - `3ds2_flow_` : 「概要・処理フロー・画面フロー」
  - `3ds2_ab_`   : 「自動課金決済」
  - `3ds2_np_`   : 「通常決済」（後で作成予定）
  - `3ds2_abchg_`: 「自動課金情報変更、停止」（後で作成予定）
- `nn`：章番号（2桁）
- `slug`：章の短い説明

このフォルダは上記ルールで、既存の `01_overview.md` 等と衝突しないファイル名に統一しています。
