# 02 自動課金初回決済（商品登録なし）仕様（P.6〜P.8）

> 出典：『クレジットカード トークン方式 3Dセキュア2.0(自動課金決済) 接続仕様書』  
> 目的：PDF内容を「漏れなく」実装で参照しやすいMarkdownに整理（ページ番号付き）  
> 注意：本資料は「自動課金決済」と「トークン作成/3DS2認証」の接続仕様を扱います。

---

## 接続先（P.6）
- URL：`https://credit.j-payment.co.jp/gateway/gateway_token.aspx`
- **接続元IP認証**あり：加盟店サーバ（固定IP）からリクエストすること。  
  ブラウザから送ると **ER003（送信元IPエラー）** が発生。

## リクエスト方式（P.6）
- GET または POST

> ※決済には **トークン作成** と **3Dセキュア2.0認証** が必須。  

---

## リクエストパラメータ（P.6〜P.7）

|項目|フィールド|詳細|必須|形式/制約|
|---|---|---|---:|---|
|店舗ID|aid|契約時に発行される店舗のID|〇|半角数字（6）|
|ジョブタイプ|jb|決済処理方法。`CAPTURE`（仮実同時売上）|〇|半角英字|
|決済結果返信方法|rt|`0`=キックバック / `1`=レスポンス|〇|半角数字（1）|
|店舗オーダー番号|cod|オーダー識別用の任意パラメータ| |50バイト以内|
|トークン情報|tkn|トークン（トークン作成処理で取得）|〇|半角英数記号|
|メールアドレス|em|メールアドレス（※2）| |半角英数（254）|
|電話番号|pn|電話番号（※2）| |半角数字（15）|
|商品金額|am|初回決済時の商品金額（※1）|〇|半角数字|
|税金額|tx|初回決済時の税金額（※1）|〇|半角数字|
|送料|sf|初回決済時の送料（※1）|〇|半角数字|
|課金周期|actp|`2`毎週 / `3`隔週 / `4`毎月 / `5`隔月 / `6`3ヶ月 / `7`6ヶ月 / `8`毎年|〇|半角数字（1）|
|自動課金金額|acam|2回目以降の金額|〇|半角数字|
|自動課金税額|actx|2回目以降の税額|〇|半角数字|
|自動課金送料|acsf|2回目以降の送料|〇|半角数字|
|課金日指定|ac1|`0`未指定 / `1~30` / `99`末日課金| |半角数字（1〜2）|
|課金停止回数指定|ac3|`0`指定なし / `1~99`| |半角数字（1〜2）|
|課金開始日指定|ac4|指定日に自動課金の1回目が課金| |日付 `YYYY/MM/DD`|
|課金終了日指定|ac5|指定日の直前の課金周期で課金後停止| |日付 `YYYY/MM/DD`|
|お試し期間タイプ|trtp|`2`日数 / `3`期限日 / `4`月数| |半角数字（1）|
|お試し日数|tr1|`0`なし / `1~999`（trtp=2時必須）| |半角数字|
|お試し月数|tr2|`0`なし / `1~24`（trtp=4時必須）| |半角数字|
|お試し期限|tr3|`YYYY/MM/DD`（trtp=3時必須）| |日付|
|お試し金額|tram|お試し終了時の決済金額（trtp指定時必須）| |半角数字|
|お試し税額|trtx|お試し終了時の決済税額| |半角数字|
|お試し送料|trsf|お試し終了時の決済送料| |半角数字|
|その他データ|（key=value）|任意（※3）| |1024バイト以内|

---

## 注記（P.8）
- **※1**：`【決済金額＋税金額＋送料】` の値がカード会社に送信されます。  
  商品金額に送料・税額を含む場合、`tx` と `sf` は `0` を指定。
- **※2**：メールアドレス `em` と電話番号 `pn` の **どちらかが必須**。
- **※3**：「改行コード」「スペース」を送信する場合は、必ずエンコード処理を行うこと。
