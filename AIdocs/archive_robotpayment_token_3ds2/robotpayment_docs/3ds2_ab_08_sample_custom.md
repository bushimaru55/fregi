# 08 カスタマイズ方式 実装方法（サンプル）（P.17〜P.18）

> 出典：『クレジットカード トークン方式 3Dセキュア2.0(自動課金決済) 接続仕様書』  
> 目的：PDF内容を「漏れなく」実装で参照しやすいMarkdownに整理（ページ番号付き）  
> 注意：本資料は「自動課金決済」と「トークン作成/3DS2認証」の接続仕様を扱います。

> 加盟店画面内にカード入力欄を配置し、`CPToken.TokenCreate` でトークン化します。

## 主要呼び出し（P.18）
- `CPToken.TokenCreate({ aid, cn, ed, fn, ln, (cvv, md, pt) }, execAuth)`
- `ThreeDSAdapter.authenticate(...)` → 成功時に決済フォーム送信
- 決済実行前にカード情報をクリア（サンプル通り）
