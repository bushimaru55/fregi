# 決済処理（商品登録なし）仕様

## 概要
- 「商品登録なし」の場合は **【商品金額、税金額、送料】の合計値** をリクエストします。
- 決済金額の指定を動的に行えるため、ショッピングカート等との連携で利用します。

## 接続先
- https://credit.j-payment.co.jp/gateway/gateway_token.aspx

### 接続元IP認証（重要）
- 接続元IP認証を行っているため、**加盟店サーバ（固定IP）から** リクエストします。
- エンドユーザーのブラウザからリクエストした場合、ER003（送信元IPエラー）が発生します。

## リクエスト方式
- GET もしくは POST

## 前提
- 決済には **トークン作成** と **3Dセキュア2.0認証** が必須です（詳細は別セクション）。

## リクエストパラメータ
|項目|フィールド|詳細|必須|形式/制約|
|---|---|---|---|---|
|店舗ID|aid|契約時に発行される店舗のID|〇|半角数字（6）|
|決済処理方法（ジョブタイプ）|jb|「AUTH」（仮売上）/「CAPTURE」（仮実同時売上）|〇|半角英字|
|決済結果返信方法|rt|「0」キックバック / 「1」レスポンス|〇|半角数字（1）|
|店舗オーダー番号|cod|任意（オーダー識別）| |50バイト以内|
|トークン情報|tkn|トークン（トークン作成処理で取得）|〇|半角英数記号|
|メールアドレス|em|メールアドレス（※4）| |半角英数（254）|
|電話番号|pn|電話番号（※4）| |半角数字（15）|
|商品金額|am|商品金額（※2）|〇|半角数字|
|税金額|tx|税金額（※2）|〇|半角数字|
|送料|sf|送料（※2）|〇|半角数字|
|その他データ|（任意）|key=value形式で任意のパラメータ（※3）| |1024バイト以内|

### 注記
- ※1 AUTH（仮売上）指定時：決済後90日以内に実売上が必要。CAPTURE（仮実同時売上）指定時：決済時に自動的に実売上。
- ※2 【決済金額＋税金額＋送料】がカード会社に送信される。商品金額に送料・税額を含む場合は tx / sf は 0 を指定。
- ※3 改行コード・スペースを送信する場合は必ずエンコード処理。
- ※4 メールアドレス、電話番号の **どちらかが必須**。
