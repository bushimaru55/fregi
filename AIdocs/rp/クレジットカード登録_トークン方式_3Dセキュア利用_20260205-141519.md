# クレジットカード登録（トークン方式 / 3Dセキュア利用）
- 仕様書: billing-robo-apispec（請求管理ロボのAPI仕様書）
- エンドポイント: `POST /api/v1.0/billing_payment_method/credit_card_token`
- 備考: 3Dセキュア未対応版は別仕様書参照（本ページは3Dセキュア対応）

## 概要
請求先登録更新後、レスポンスで返却された店舗オーダー番号をリクエストに加え、ROBOT PAYMENT決済システムにリクエスト送信する際に、ユーザーが入力するクレジットカード番号を別文字列（トークン）に置き換えて通信することで、情報漏洩リスクを軽減しつつ請求先の決済情報へクレジットカード情報を登録します。

決済方式は次の2種類です。

- **ポップアップ方式**: JavaScriptでブラウザ内に決済画面をポップアップ表示し、カード情報をトークンへ置き換えて加盟店へ返却 → 登録処理。
- **カスタマイズ方式**: 加盟店サイト内でカード情報入力 → JavaScriptで弊社サーバーへ送信しトークン化 → 加盟店へ返却 → 登録処理。

> トークン決済/3Dセキュア導入は**事前に弊社側設定が必要**（希望時はサポートへ連絡）。

---

## 3Dセキュア導入に関する留意点
- 原則 **2025年3月末までに実装が必要**。
- 3Dセキュア未導入企業は、カード会社方針により**カード契約強制解約**となる可能性がある。
- **検証環境では検証不可**（仕様上）。
- 既にトークン決済利用中でも、3Dセキュア導入には**事前の弊社側設定**が必要。

### 3Dセキュア2.0概要
EMVCo（国際カードブランド6社の団体）による3Dセキュア改良版。本人認証を行いつつ、リスクが低ければ認証画面をスキップする**フリクションレスフロー**が導入されています。ネット決済でも本人確認が可能となり、加盟店の**なりすまし被害等のチャージバック抑止**に寄与します。

## 実装方式サンプル1（ポップアップ方式）

### 1) JavaScriptのインポート
```html
<head>
  <meta charset="utf-8" />
  <script type="text/javascript" src="https://credit.j-payment.co.jp/gateway/js/jquery.js"></script> <!--※1-->
  <script type="text/javascript" src="https://credit.j-payment.co.jp/gateway/js/CPToken.js"></script><!--※2-->
  <script type="text/javascript" src="https://credit.j-payment.co.jp/gateway/js/EMV3DSAdapter.js"></script><!--※3-->
</head>
```
- ※1 加盟店側でjQuery読み込み済みなら不要
- ※2 トークン決済に必要なJS
- ※3 3Dセキュア2.0に必要なJS

### 2) トークン格納フィールド & ポップアップ用要素
`CPToken.CardInfo()` 呼び出しにより、`<form id="mainform">` の `<input id="tkn">` にトークンが格納されます。

```html
<form id="mainform">
  <input id="tkn" name="tkn" type="hidden" value="">
  <div id="CARD_INPUT_FORM"></div>
  <div id="EMV3DS_INPUT_FORM"></div>
  <input type="button" value="購入する" onclick="doPurchase()"/>
</form>
```

### 3) トークン作成処理
```javascript
function doPurchase() {
  CPToken.CardInfo({
    aid: '000000' // 店舗ID
  }, execAuth);    // 3Dセキュア認証用コールバック
}
```

### 4) 3Dセキュア2.0認証（商品登録なし）
```javascript
function execAuth(resultCode, errMsg) {
  if (resultCode != "Success") {
    window.alert(errMsg);
  } else {
    ThreeDSAdapter.authenticate({
      tkn: $("#tkn").val(),
      aid: '000000',
      am: 0,  // am/tx/sf は0固定
      tx: 0,
      sf: 0,
      em: 'sample@sample.com', // 請求先(部署)に登録されているメールと同値必須
      pn: '0300000000'         // 請求先(部署)に登録されている電話と同値必須（ハイフン除去）
    }, execPurchase);
  }
}
```

### 5) 決済処理実行（加盟店サーバーへ送信）
```javascript
function execPurchase(resultCode, errMsg) {
  if (resultCode != "Success") {
    window.alert(errMsg);
  } else {
    window.alert("PAYMENT SUCCESS!!");
    $("#mainform").submit();
  }
}
```

### ポップアップ方式：トークン生成関数
```text
CPToken.CardInfo(
  { 送信するJSONパラメータ },
  コールバック関数
);
```

#### パラメータ
| 名前 | 概要 | 桁数 | 種別 | 必須 |
|---|---|---:|---|---|
| aid | 店舗ID（契約時に発行された決済システム店舗ID） | 6 | 数値 | 必須 |

## 実装方式サンプル2（カスタマイズ方式）

### 1) JavaScriptのインポート
```html
<head>
  <meta charset="utf-8" />
  <script type="text/javascript" src="https://credit.j-payment.co.jp/gateway/js/jquery.js"></script> <!--※1-->
  <script type="text/javascript" src="https://credit.j-payment.co.jp/gateway/js/CPToken.js"></script><!--※2-->
  <script type="text/javascript" src="https://credit.j-payment.co.jp/gateway/js/EMV3DSAdapter.js"></script><!--※3-->
</head>
```
- ※1 加盟店側でjQuery読み込み済みなら不要
- ※2 トークン決済に必要なJS
- ※3 3Dセキュア2.0に必要なJS

### 2) 入力フォーム（トークン格納 & 3DS表示領域）
`CPToken.TokenCreate()` 呼び出しにより、`<input id="tkn">` にトークンが格納されます。

```html
<form id="mainform">
  カード番号：
  <input type="text" value="" name="cn" id="cn" />
  カード有効期限：
  <input type="text" value="" name="ed_year" id="ed_year" /> /
  <input type="text" value="" name="ed_month" id="ed_month" />
  カード名義人：
  <input type="text" value="" name="fn" id="fn" />
  <input type="text" value="" name="ln" id="ln" />

  <input id="tkn" name="tkn" type="hidden" value="">
  <div id="EMV3DS_INPUT_FORM"></div>

  <input type="button" value="購入する" onclick="doPurchase()"/>
</form>
```

### 3) トークン作成処理
```javascript
function doPurchase() {
  CPToken.TokenCreate({
    aid: '000000',
    cn: $("#cn").val(),
    ed: $("#ed_year").val() + $("#ed_month").val(),
    fn: $("#fn").val(),
    ln: $("#ln").val()
  }, execAuth);
}
```

### 4) 3Dセキュア2.0認証（商品登録なし）
```javascript
function execAuth(resultCode, errMsg) {
  if (resultCode != "Success") {
    window.alert(errMsg);
  } else {
    ThreeDSAdapter.authenticate({
      tkn: $("#tkn").val(),
      aid: '000000',
      am: 0,
      tx: 0,
      sf: 0,
      em: 'sample@sample.com',
      pn: '0300000000'
    }, execPurchase);
  }
}
```

### 5) 決済処理実行（カード情報消去→サーバ送信）
```javascript
function execPurchase(resultCode, errMsg) {
  if (resultCode != "Success") {
    window.alert(errMsg);
  } else {
    // カード情報を消去
    $("#cn").val("");
    $("#ed_year").val("");
    $("#ed_month").val("");
    $("#fn").val("");
    $("#ln").val("");

    $("#mainform").submit();
  }
}
```

> 注意: 入力されたカード情報の削除、および加盟店サーバへカード情報を送信しない実装が必要。

### カスタマイズ方式：トークン生成関数
```text
CPToken.TokenCreate(
  { 送信するJSONパラメータ },
  コールバック関数
);
```

#### パラメータ
| 名前 | 概要 | 桁数 | 種別 | 必須 |
|---|---|---:|---|---|
| aid | 店舗ID（契約時に発行された決済システム店舗ID） | 6 | 数値 | 必須 |
| cn | カード番号 | 16 | 数値 | 必須 |
| ed | カード有効期限（YYMM形式） | 4 | 数値 | 必須 |
| fn | カード名義（姓） | 50 | 文字列 | 必須 |
| ln | カード名義（名） | 50 | 文字列 | 必須 |
| cvv | セキュリティコード | 3または4 | 数値 | （セキュリティコード利用オプション時） |
| md | 支払い方法（「10」一括払い） | 2 | 定型 | 必須 |

## クレジットカード登録 API

### リクエスト
- Path: `/api/v1.0/billing_payment_method/credit_card_token`
- Method: `POST`
- Content-Type: `application/json`
- Encode: `UTF-8`

#### Parameters（トップレベル）
| 名前 | 概要 | 桁数 | 種別 | 必須 |
|---|---|---:|---|---|
| user_id | ユーザID（管理画面へのログインID） | 100 | メール形式 | 必須 |
| access_key | アクセスキー | 100 | 半角英数 | 必須 |
| billing_code | 請求先コード | 20 | 半角英数 + 記号 | 必須 |
| billing_payment_method_number | 決済情報番号 | 18 | 数値 | （必須）^1 |
| billing_payment_method_code | 決済情報コード | 20 | 半角英数 + 記号 | （必須）^1 |
| token | トークン情報 |  | 半角英数 | 必須 |
| email | メールアドレス（請求先(部署)登録値と同値必須、両端スペース除去） | 100 | メール形式 |  |
| tel | 電話番号（請求先(部署)登録値と同値必須、両端スペース除去） | 15 | 数値 |  |

> ^1: `billing_payment_method_number` と `billing_payment_method_code` は「同時指定不可」条件あり（個別エラー参照）。

### レスポンス
- Type: `application/json`
- Encode: `UTF-8`

#### Fields
| 名前 | 概要 | 型 |
|---|---|---|
| error | エラー（正常時はnull） | object \| null |

`error` オブジェクト:
| 名前 | 概要 | 型 |
|---|---|---|
| code | エラーコード | int |
| message | エラーメッセージ | string |

## 使用例

### リクエスト例
```json
{
  "user_id": "sample@robotpayment.co.jp",
  "access_key": "xxxxxxxxxxxxxxxx",
  "billing_code": "billing_code",
  "billing_payment_method_number": 1,
  "token": "XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
  "email": "sample@sample.com",
  "tel": "0300000000"
}
```

### レスポンス例
Status: `200 OK`
```json
{
  "error": {
    "code": "ER000",
    "message": "Credit card payment error."
  }
}
```
