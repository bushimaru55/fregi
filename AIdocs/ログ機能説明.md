# 決済フロー用ログ機能

## 概要

決済フローのテスト時にエラーが発生した場合、原因を追跡できるように詳細なログを記録する機能を実装しました。

## ログファイルの場所

ログファイルは以下の場所に保存されます：

```
storage/logs/contract-payment-YYYY-MM-DD.log
```

例：`storage/logs/contract-payment-2026-01-13.log`

## ログの保持期間

ログファイルは **30日間** 保持されます。30日を超えたログファイルは自動的に削除されます。

## 記録される情報

### 1. 申込フォーム表示時 (`create`メソッド)

- URL（完全なURL）
- 選択されたプランID（`plans`パラメータ）
- 表示されたプラン数
- アクセス元IPアドレス
- ユーザーエージェント
- タイムスタンプ

### 2. 確認画面表示時 (`confirm`メソッド)

- 契約プランID
- プラン名・価格
- 会社名（マスク済み：最初の3文字のみ表示）
- メールアドレス（マスク済み：最初の3文字のみ表示）
- アクセス元IPアドレス
- ユーザーエージェント
- タイムスタンプ

### 3. 決済処理時 (`store`メソッド)

#### 処理開始
- 契約プランID
- アクセス元IPアドレス
- ユーザーエージェント

#### 契約作成完了
- 契約ID
- 契約プランID
- ステータス

#### 決済情報作成完了
- 決済ID
- 契約ID
- 伝票番号（orderid）
- 金額
- ステータス

#### F-REGI API呼び出し
- 決済ID
- 伝票番号
- APIパラメータ
- 決済タイプ（一回限り/月額）
- 環境（test/prod）

#### F-REGI API結果
- 決済ID
- 結果（OK/NG）
- 発行番号（settleno）
- エラーメッセージ（エラー時）

#### 決済処理完了
- 決済ID
- 契約ID
- 伝票番号
- 発行番号
- 決済画面URL
- 処理時間（ミリ秒）

### 4. エラー発生時

- エラーメッセージ
- スタックトレース
- ファイル名・行番号
- アクセス元IPアドレス
- 処理時間（ミリ秒）

## ログの確認方法

### ローカル環境（Docker）

#### 1. Dockerコンテナ内から確認

```bash
docker compose exec app bash
cd /var/www
tail -f storage/logs/contract-payment-$(date +%Y-%m-%d).log
```

#### 2. 最新のログを確認

```bash
docker compose exec app bash -c "cd /var/www && tail -100 storage/logs/contract-payment-$(date +%Y-%m-%d).log"
```

#### 3. エラーログのみを確認

```bash
docker compose exec app bash -c "cd /var/www && grep -i error storage/logs/contract-payment-$(date +%Y-%m-%d).log"
```

#### 4. 特定の決済IDで検索

```bash
docker compose exec app bash -c "cd /var/www && grep 'payment_id.*123' storage/logs/contract-payment-*.log"
```

### 本番環境（Plesk）

**重要**: 本番環境ではSSH/CLIが使えないため、Pleskファイルマネージャーからログファイルを確認します。

#### ログファイルの場所

```
/httpdocs/webroot/billing/app/storage/logs/contract-payment-YYYY-MM-DD.log
```

**例**: `/httpdocs/webroot/billing/app/storage/logs/contract-payment-2026-01-21.log`

#### 確認手順

1. **Pleskにログイン**
   - Pleskコントロールパネルにアクセス

2. **ファイルマネージャーを開く**
   - 左メニューから「ファイルマネージャー」をクリック

3. **ログディレクトリに移動**
   - `/httpdocs/webroot/billing/app/storage/logs/` に移動
   - または、パスバーに直接入力して移動

4. **ログファイルを開く**
   - 日付付きのログファイル（例: `contract-payment-2026-01-21.log`）を選択
   - 「編集」または「表示」をクリックして内容を確認

#### エラーログの検索方法

**最新のエラーログを確認**:
- ファイルの末尾（最新のエラー）を確認
- ファイル内で「error」や「エラー」で検索（Ctrl+F / Cmd+F）

**オプション製品APIエラーの確認**:
- 以下のキーワードで検索：
  - `オプション製品取得エラー`
  - `getOptionProducts`
  - `contract_plan_id`

**特定の日付のログを確認**:
- ファイル名に日付が含まれているため、該当日付のファイルを開く

#### ログファイルのパーミッション確認

ログが記録されない場合は、パーミッションを確認してください：

- **ディレクトリ**: `storage/logs/` のパーミッションを `775` に設定
- **ファイル**: ログファイルのパーミッションを `664` に設定

Pleskファイルマネージャーでパーミッションを変更できます。

## ログレベル

- `info`: 正常な処理フロー（本番環境でも記録されます）
- `error`: エラー発生時
- `warning`: 警告（現在は使用していません）

**重要**: `contract_payment`チャンネルは、本番環境でも`info`レベルのログを記録するように設定されています。これにより、正常な処理フローも追跡可能で、トラブルシューティング時に必要な情報が記録されます。

## セキュリティ上の注意

- 個人情報（会社名、メールアドレス）はマスクして記録されます
- パスワードやカード情報は記録されません
- IPアドレスとユーザーエージェントは記録されます（トラブルシューティング用）

## トラブルシューティング

### ログファイルが見つからない場合

1. ログディレクトリの権限を確認
```bash
docker compose exec app bash -c "cd /var/www && ls -la storage/logs/"
```

2. ログファイルが作成されているか確認
```bash
docker compose exec app bash -c "cd /var/www && ls -lh storage/logs/contract-payment-*.log"
```

### ログが記録されない場合

#### ローカル環境（Docker）

1. ログチャンネルの設定を確認
```bash
docker compose exec app bash -c "cd /var/www && php artisan config:show logging.channels.contract_payment"
```

2. ログレベルを確認
```bash
docker compose exec app bash -c "cd /var/www && php artisan config:show logging.channels.contract_payment.level"
```

#### 本番環境（Plesk）

1. **ログディレクトリのパーミッションを確認**
   - Pleskファイルマネージャーで `/httpdocs/webroot/billing/app/storage/logs/` のパーミッションを確認
   - ディレクトリのパーミッションが `775` になっているか確認

2. **ログファイルが作成されているか確認**
   - ログディレクトリ内に日付付きのログファイルが存在するか確認
   - ファイルが存在しない場合、アプリケーションにアクセスしてログファイルが作成されるか確認

3. **Config Cacheをクリア**（必要に応じて）
   - `/httpdocs/webroot/billing/app/bootstrap/cache/config.php` を削除
   - 管理画面にアクセスしてconfig cacheを再生成

4. **エラーログを確認**
   - `/httpdocs/webroot/billing/app/storage/logs/laravel.log` を確認
   - ログ記録に関するエラーがないか確認
