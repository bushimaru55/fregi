# 請求管理ロボ API シーケンス（請求先登録〜商品参照）

**出典**: 請求管理ロボ_APIシーケンス図.pdf（改訂履歴: 2017/6/15 新規作成〜2020/8/31 新規API追加）

---

## 共通前提

- **範囲**: 請求先登録から商品参照までの一連の流れ。
- **編集**: 各オブジェクト単位でリクエスト可能。
- **停止・削除**: 請求先・請求先部署・請求情報・商品オブジェクトに対してリクエスト可能。
- **親子関係**: 請求先は親概念。請求先を停止・削除すると、子要素（請求先補助科目、請求先部署、請求先部署補助科目、決済情報、請求情報）も停止・削除される。

---

## 共通で使うAPIの順序（概念）

1. **請求先登録更新（複数）** `POST /api/v1.0/billing/bulk_upsert`
   - 請求先部署のデフォルト決済手段に決済情報を紐付ける場合は `billing.individual.payment_method_code` を送信。
   - 請求先部署紐付けまたは請求情報登録で必要なため、決済情報コード `billing.payment.code` を指定する。
2. **レスポンス**
3. （決済手段により **クレジットカード登録** や **決済システム連携** がここに入る場合あり → 下記「決済手段別」参照）
4. **請求情報登録更新（複数）** `POST /api/v1.0/demand/bulk_upsert`
   - 請求情報（demand）に属するパラメータ。
   - ①の請求先コード（`billing_code`）、請求先部署番号（`billing_individual_number`）またはコード（`billing_individual_code`）、決済情報コード（`payment_method_code`）を指定。
   - 商品を指定する場合: 商品マスタから呼び出し。商品コード（`item_code`）を指定すると商品マスタの値が反映される（請求情報登録で指定した値が優先）。
5. **レスポンス**
6. **請求管理ロボ側のシステム処理**
   - 売上: 請求情報の**売上計上日**を元に作成（請求書手動発行時または売上作成バッチ）。
   - 請求書: 請求情報の**請求書発行日**を元に作成（請求書手動発行時または請求書自動発行バッチ）。
7. 以降、参照系・入金・消込など（決済手段により入金の発生タイミングが異なる）。

## 決済手段別のシーケンス概要

### 1. 銀行振込

- ①請求先登録更新 → ②レスポンス → ③請求情報登録更新 → ④レスポンス → 請求書発行・請求書送付 → 入金 → ⑤請求書参照 → ⑥レスポンス → ⑦入金参照 → ⑧入金無効削除 → ⑨レスポンス → ⑩入金登録更新（0:自動消込選択時）→ ⑪レスポンス → ⑫消込 → ⑬消込取消／消込結果参照／消込結果明細参照 → ⑭レスポンス → 請求先部署参照 → 請求情報参照 → 請求書明細参照 → 請求書参照 → 決済情報参照 → 商品参照（各レスポンスあり）。

### 2. クレジットカード

- ①請求先登録更新 → ②レスポンス → **③決済システムが提供するJavaScriptを読み込み、クレジットカード情報を送信** → **④トークンを受け取る** → **⑤クレジットカード登録（トークン方式）** `POST /api/v1.0/billing_payment_method/credit_card_token`（決済情報と④のトークン情報を送信）→ ⑥レスポンス → ⑦請求情報登録更新 → ⑧レスポンス → 売上作成・請求書発行 → **請求書の決済期限当日に決済システムへ決済データを送信** → 決済キックバック → ⑨請求書参照 → ⑩レスポンス → 入金参照・入金無効削除・入金登録更新（自動消込時は消込実行）→ 以降、請求先部署参照・請求情報参照・請求書明細参照・請求書参照・決済情報参照・商品参照。

### 3. バンクチェック（バーチャル口座）

- ①請求先登録更新 → 決済システムへバンクチェック口座情報取得をリクエスト → ②レスポンス → ③請求情報登録更新 → ④レスポンス → 売上・請求書作成 → 請求書送付 → バーチャル口座に入金 → 決済キックバック → 請求書参照以降は入金参照・入金無効削除・入金登録更新・消込〜各種参照。

### 4. RP口座振替

- ①請求先登録更新 → 決済システムへCP口座情報を送信 → ②レスポンス → ③請求情報登録更新 → ④レスポンス → 売上・請求書作成 → **請求書発行時に口座振替情報を送信** → 口座振替処理・振替結果 → 決済キックバック（日次バッチ 11時・13時・16時）→ ⑤請求書参照以降、入金参照・入金登録更新・消込〜各種参照。

### 5. RL・その他口座振替

- ①請求先登録更新 → ②レスポンス → ③請求情報登録更新 → ④レスポンス → 売上・請求書作成 → **⑤請求書発行後、口座振替データ（CSV）を口座振替業者システムにインポート** → 口座振替処理・振替結果 → **⑥口座振替後、口座振替データ（CSV）を経理のミカタにインポート** → ⑦請求書参照以降、入金参照・入金登録更新・消込〜各種参照。

### 6. コンビニ払込票（A4・ハガキ）

- ①請求先登録更新 → ②レスポンス → ③請求情報登録更新 → ④レスポンス → 売上・請求書作成 → **請求書発行時にコンビニ決済情報を送信** → 翌営業日にコンビニ払込票送付 → 入金 → 決済キックバック（入金があった日の翌日15:30頃）→ ⑤請求書参照以降、入金参照・入金登録更新・消込〜各種参照。

## クレジットカード（トークン方式）のポイント

- カード情報は**決済システム提供のJavaScript**で送信し、**トークン**を取得する。
- 取得したトークンと**決済情報**を `POST /api/v1.0/billing_payment_method/credit_card_token` に送信してクレジットカードを登録する。
- 請求書の決済期限当日に、決済システムへ決済データが送信され、決済キックバックで結果が返る。
