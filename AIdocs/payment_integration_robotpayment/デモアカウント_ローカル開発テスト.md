# ROBOT PAYMENT デモアカウントでローカル開発テスト

デモアカウントを利用して、ローカル環境から申込〜決済までの開発・テストを行う手順です。

---

## 0. 重要：2種類の管理画面の違い

ROBOT PAYMENT には **2種類の管理画面** があり、設定項目が異なります。

| システム | URL | 主な用途 |
|----------|-----|----------|
| **請求管理ロボ** | https://demo.billing-robo.jp/login | API（user_id・access_key）、請求先・決済情報参照、**決済結果通知URL**の登録、Webhook署名キーなど |
| **決済システムコントロールパネル** | https://credit.j-payment.co.jp/cp/SignIn.aspx | **店舗ID（aid）** の確認、**決済データ送信元URL（リファラー）**・送信元IP の設定 |

- **店舗ID（aid）** は **請求管理ロボには表示されません**。決済システムコントロールパネル（credit.j-payment.co.jp）にログインし、画面内に記載の「店舗ID」を確認してください。
- **リファラーエラー**（「店舗設定からリファラーURLを設定してください」）を解消するには、**決済システムコントロールパネル** の「PC用決済フォーム設定」→「決済データ送信元URL」にローカルURLを登録する必要があります。
- デモ契約で **credit.j-payment.co.jp へのログイン権限** が別途あるかは契約内容によります。ない場合は ROBOT PAYMENT サポートへ「店舗IDの確認」と「ローカル開発用の決済データ送信元URL登録」を依頼してください。
  - **カスタマーサポート**: TEL 03-4405-0665（平日9:00–18:00）、Mail support@billing-robo.jp

（出典: [請求管理ロボ API 仕様 クレジットカード登録(トークン方式 3Dセキュア利用)](https://apispec.billing-robo.jp/public/billing_payment_method/credit_card_token.html)、[MyASP マニュアル ROBOT PAYMENT決済](https://docs.myasp.jp/19489/)）

## 2. ローカルで必要な準備

### 2.1 店舗ID（aid）の取得

店舗IDは **決済システムコントロールパネル** で確認します（請求管理ロボの「API接続詳細」には表示されません）。

1. **決済システムコントロールパネル** にログインする。  
   URL: https://credit.j-payment.co.jp/cp/SignIn.aspx  
   （デモ契約でこのCPへのログイン権限が無い場合は、ROBOT PAYMENT サポートに店舗IDの照会を依頼してください。）
2. コントロールパネル画面内に記載の **店舗ID（6桁の半角数字）** を確認する。
3. その値を `.env` の `ROBOTPAYMENT_STORE_ID` に設定する。

（デモ用の店舗IDがサポートから別途案内されている場合は、その値をそのまま使用してください。MyASP マニュアルでは「ROBOT PAYMENTコントロールパネル側に記載されている店舗IDを入力」とされています。）

### 2.2 通知URLについて（ローカル）

- RP の「決済結果通知」「自動課金結果通知」は **RP サーバから当システムへ GET で呼び出されます**。
- ローカル（localhost）はインターネットから直接アクセスできないため、次のいずれかが必要です。
  - **ngrok 等**でローカルサーバを一時的に公開し、発行された URL（例: `https://xxxx.ngrok.io/billing/public/api/robotpayment/notify-initial`）を RP デモ管理画面の「決済結果通知設定」に登録する。
  - または、**通知までは検証せず**「申込 → 確認 → 決済ページ表示・トークン取得・gateway 送信」までをローカルで確認し、通知の検証はステージング環境で行う。

### 2.3 ゲートウェイURL（デモ用）

- 通常は `https://credit.j-payment.co.jp/gateway/gateway_token.aspx` のまま利用します。
- デモ環境で **別のゲートウェイURL** が案内されている場合は、`.env` の `ROBOTPAYMENT_GATEWAY_URL` をその値に変更してください。

### 2.4 決済データ送信元URL（リファラー）（必須：ローカルでカード入力する場合）

- 決済ページで RP の JS（CPToken.js）が動作する際、**リファラー（参照元URL）の検証**が行われます。
- 「店舗設定からリファラーURLを設定してください」とアラートが出る場合は、**決済システムコントロールパネル** で「決済データ送信元URL」にローカル環境のURLを登録してください（請求管理ロボではなく、**credit.j-payment.co.jp** 側の設定です）。

**公式API仕様に記載の設定場所（請求管理ロボ API 仕様書「クレジットカード登録(トークン方式 3Dセキュア利用)」より）:**

> 設定が完了後、コントロールパネルより以下の設定をお願い致します。  
> 1. 「**設定**」→「**決済システム**」  
> 2. 「決済ゲートウェイ＆CTI決済設定」→「決済データ送信元IP」  
> 3. 「**PC用決済フォーム設定**」→「**決済データ送信元URL**」

**手順の例:**

1. **決済システムコントロールパネル** にログインする。  
   https://credit.j-payment.co.jp/cp/SignIn.aspx
2. **「設定」→「決済システム」** のページに移動する。
3. **「PC用決済フォーム設定」** の **「決済データ送信元URL」** を開く。
4. 次を **1件以上** 登録する（実際にブラウザで開いているURLのオリジンに合わせる）。
   - `http://localhost:8080`
   - または `http://127.0.0.1:8080`
   - サブディレクトリで運用している場合は `http://localhost:8080/billing` のようにベースURLで登録できるか、RP の仕様に従う。
5. 保存後、ブラウザで決済ページを再読み込みし、再度カード入力を試す。

※ デモ契約で credit.j-payment.co.jp にログインできない場合は、ROBOT PAYMENT サポート（support@billing-robo.jp）に「ローカル開発用の決済データ送信元URL（例: http://localhost:8080）の登録」を依頼してください。  
※ 登録できる形式（末尾スラッシュの有無・パスまで必要かなど）は RP 管理画面の表示に従ってください。

## 3. .env 設定例（デモでローカル開発する場合）

```env
ROBOTPAYMENT_ENABLED=true
# デモ管理画面で確認した店舗ID（6桁）を設定
ROBOTPAYMENT_STORE_ID=（デモ管理画面で確認した値）
# デモで発行されたAPI接続用アクセスキー（案内されている場合に設定）
ROBOTPAYMENT_ACCESS_KEY=（発行時に共有されたアクセスキー）
ROBOTPAYMENT_GATEWAY_URL=https://credit.j-payment.co.jp/gateway/gateway_token.aspx
ROBOTPAYMENT_COMPANY_ID=1
# ローカルで通知を受ける場合は ngrok 等の公開URLを RP 管理画面に登録し、ここは参照用で同じ値を書いても可
ROBOTPAYMENT_NOTIFY_INITIAL_URL=
ROBOTPAYMENT_NOTIFY_RECURRING_URL=
```

反映:

```bash
cd app
php artisan config:clear
```

## 4. ローカル開発テストの実行手順

1. **セットアップ（初回または DB リセット時）**
   ```bash
   cd app
   php artisan robotpayment:setup-local
   ```

2. **アプリ起動**
   ```bash
   php artisan serve
   ```
   （Docker で運用している場合は `docker compose up` 等で起動）

3. **ブラウザで確認**
   - 申込フォーム（例: http://127.0.0.1:8000 または http://localhost:8080/billing）を開く。
   - プラン選択・必須項目入力 → 「確認する」→ **決済ページ**へ遷移することを確認。
   - カード番号は **RP から案内されたテスト用カード番号** があればそれを使用する（未案内の場合は要・RP 問い合わせ）。

4. **決済〜通知まで検証する場合**
   - ngrok 等で公開URLを用意し、RP デモ管理画面に「決済結果通知URL」「自動課金結果通知URL」を登録。
   - 決済実行後、RP から通知が飛び、当システムの `payment_events` にレコードが保存されることを確認。
