# 02 Parameter Mapping（自動課金：初回合算 → 次月以降月額のみ）

本ドキュメントは、当社要件の中核である **「初回合算（買い切り＋月額）」** を、**ユーザー操作1回で実現**するためのパラメータ指定を確定する。

一次情報：
- 自動課金初回決済（商品登録なし）：`AIdocs/robotpayment_docs/3ds2_ab_02_initial_no_item.md`（P.6〜P.8）
- 自動課金初回決済（商品登録あり）：`AIdocs/robotpayment_docs/3ds2_ab_03_initial_with_item.md`
- 3DS認証（商品登録なし/あり）：`AIdocs/robotpayment_docs/3ds2_np_12_3ds_auth_no_item_spec.md` / `..._with_item_spec.md`

---

## 2.1 基本方針（当社確定）
- **初回金額**は `am/tx/sf` に設定する（合算可）
- **2回目以降金額**は `acam/actx/acsf` に設定する（＝月額のみ）
- 課金周期は `actp` を必須指定（例：毎月=4）
- 次回以降の課金日を固定する場合は `ac1`（課金日）と `ac4`（課金開始日）を併用する

この設計により、**エンドユーザーは初回の決済操作1回のみ**で、次月以降はRPが自動課金する。

---

## 2.2 パラメータ一覧（商品登録なし：自動課金初回決済）
接続先：`https://credit.j-payment.co.jp/gateway/gateway_token.aspx`  
- **接続元IP認証あり**：加盟店サーバ（固定IP）から送ること（ブラウザ直送不可）。  
  - 一次情報：`AIdocs/robotpayment_docs/3ds2_ab_02_initial_no_item.md`（ER003 送信元IPエラー）

### 2.2.1 決済共通
|区分|項目|パラメータ|当社方針|備考/根拠|
|---|---|---|---|---|
|必須|店舗ID|`aid`|必須|一次情報の必須|
|必須|ジョブタイプ|`jb`|`CAPTURE`（仮実同時売上）|一次情報|
|必須|返信方法|`rt`|`0`（キックバック）を推奨|非同期で扱いやすい。一次情報で0/1定義|
|任意|店舗オーダー番号|`cod`|必須運用（当社）|照合キー。一次情報は任意だが当社は必須|
|必須|トークン|`tkn`|必須|トークン作成処理で取得|
|必須|連絡先|`em` or `pn`|どちらか必須|一次情報：どちらか必須|

### 2.2.2 初回金額（合算）
|区分|項目|パラメータ|当社方針|備考|
|---|---|---|---|---|
|必須|初回商品金額|`am`|必須|合算金額を入れる（買い切り＋月額）|
|必須|初回税額|`tx`|原則0|税を金額に含めるなら0（一次情報注記）|
|必須|初回送料|`sf`|原則0|送料を金額に含めるなら0（一次情報注記）|

### 2.2.3 2回目以降金額（月額のみ）
|区分|項目|パラメータ|当社方針|備考|
|---|---|---|---|---|
|必須|課金周期|`actp`|毎月=`4`|一次情報：2毎週/3隔週/4毎月/…|
|必須|自動課金金額|`acam`|月額のみ|一次情報：2回目以降の金額|
|必須|自動課金税額|`actx`|原則0|月額の税を別建てなら設定|
|必須|自動課金送料|`acsf`|原則0|月額の送料を別建てなら設定|
|任意|課金日指定|`ac1`|毎月1日なら`1`|未指定なら初回決済日が課金日（一次情報）|
|任意|課金開始日|`ac4`|当社ルールで指定|指定日に自動課金の1回目が課金（一次情報）|

### 2.2.4 お試し（必要になった場合にのみ実装）
一次情報にはお試し関連（`trtp/tr1/tr2/tr3/tram/trtx/trsf`）が定義されている。必要になった時点で当社要件として採否を決め、採用時は本書へ追記する。  
- 参照：`AIdocs/robotpayment_docs/3ds2_ab_02_initial_no_item.md`（tr* の定義）

## 2.3 具体例（あなたのケース：AIチャットボット月額＋デザイン買い切り）

### 金額
- AIチャットボット（月額）：10,000円
- デザイン（買い切り）：50,000円

### 初回（合算）
- `am = 60000`
- `tx = 0`
- `sf = 0`

### 次月以降（月額のみ）
- `actp = 4`（毎月）
- `acam = 10000`
- `actx = 0`
- `acsf = 0`

## 2.4 課金日の指定ルール（当社）
当社の月額課金日は **毎月1日** を標準とする（後から変更する場合は `05_operations.md` に記載する運用手順を追加）。

### ルール
- `ac1 = 1` を指定する（毎月1日課金）
- `ac4` は「自動課金の1回目（＝初回決済の次の決済）」をいつ実行するかを制御する

#### `ac4` 決定ロジック（標準）
申込日（初回決済日）を `D` とする。

- D が「当月1日」以外の場合：`ac4 = 翌月1日`
- D が「当月1日」の場合：`ac4 = 当月1日`（＝直ちに次回課金が発生しうるため、通常は避ける。実運用では **翌月1日** に寄せる）

> 注：`ac4` を指定しない場合の挙動は一次情報に依存するため、当社は **明示指定を標準** とする。

### 例（申込日=2026/02/03）
- `ac1=1`
- `ac4=2026/03/01`

## 2.5 3DS認証で設定すべき値（重要）
3DS認証（`ThreeDSAdapter.authenticate`）では、**決済処理と同じ値**（`am/tx/sf` または `iid`）を指定する。  
- 参照：`AIdocs/robotpayment_docs/3ds2_np_12_3ds_auth_no_item_spec.md`（注記「決済処理と同じ値を指定」）

また、3DS認証の `Success` は **API呼び出し成功**であり、**認証・決済の成功を意味しない**。最終的な成否は後続の決済処理（`gateway_token.aspx`）結果で判定する。  
- 参照：同上 callback 仕様

## 2.6 実装チェックリスト（失敗しやすい点）
- [ ] ブラウザから `gateway_token.aspx` に直接POSTしていない（固定IP制約でER003）
- [ ] 初回合算は `am` に、次月以降は `acam` に設定している（逆にしない）
- [ ] 月額課金日固定の場合、`ac1` と `ac4` の指定が整合している
- [ ] 3DS認証パラメータが決済と一致している
- [ ] 通知受信で `acid` を保存している（自動課金のキー）
